## D6 状态机修订(替换设计文档 line 79–87)
```
unseen ──[勾选已练]──> practiced
unseen ──[任意错误]──> practiced                  (防御性)
practiced ──[累计答对 1 次]──> practiced          (无变化)
practiced ──[累计答对 3 次]──> mastered
practiced ──[该 id 错误次数累计 ≥ 2]──> recent_mistakes
mastered ──[答错]──> practiced
recent_mistakes ──[累计答对 1 次]──> practiced
recent_mistakes ──[累计答对 2 次]──> mastered     (跳过 practiced,直接升级)
```

错误计数重置规则:进入 mastered 时 `mistakeCount[id] = 0`。

## Parallelization Lanes(单人最大化并行)

**Dependency 表:**

| Step / Lane | Modules touched | Depends on |
|------|----------------|------------|
| Lane A: 脚手架 | App/, Model/, Persistence/, MacTrainerTests/ | — |
| Lane B: 内容录入 | Resources/data/ | — (独立轨道) |
| Lane C: Review 模块 | Review/ | Lane A (Model/ 类型) |
| Lane D: UI 模块 | Views/ | Lane A (Model/ 类型 + AppModel 接口) |
| Lane E: 集成 | App/, 跨 module | Lane A + C + D + B |

**执行顺序:**
1. Week 1 启动两个并行轨道:
   - Lane A (4 天):Xcode 项目、目录结构、Codable、AppModel 骨架、Normalization、第一个 sample 测试
   - Lane B (持续):手工录入 50 条系统级,先入 draft JSON
2. Week 2 Lane A 完成,Lane C + Lane D 启动并行:
   - Lane C (5 天):StateMachine、DistractorGenerator、KeyMatcher、各自测试
   - Lane D (5 天):ContentView、SidebarView、MiddleListView、DetailView、SearchField
   - Lane B 持续:累计 100 条
3. Week 3 Lane C + D 收尾,Lane E 启动:
   - Lane E (5 天):QuizPanelView、集成到 DetailView、AppModel 接 Review 模块
   - Lane B 持续:累计 150 条
4. Week 4 单向抽考 + 数据 180 条
5. Week 5 双向抽考 + 暗色 + 200 条 + dogfood

**关键:** Lane B(内容录入)是低认知任务,在 Lane A/C/D 感到疲劳时切换。**这是单人"并行"的真实意义。**

**冲突标记:** Lane C 和 Lane D 不共享文件,可以真正物理并行(比如两个 worktree)。Lane A 完成后 Lane C/D 也可分 worktree。Lane E 必须在所有 lane 合并后做。

## Test Strategy(D5 锁定)

**框架:** Swift Testing (`import Testing`)

**4 块必测(覆盖率目标 70% for Model/ and Review/):**

1. **JSON 加载 + 边界检查** (`Model/ShortcutTests.swift`)
   - 加载 sample.json,所有 10 条反序列化成功
   - 加载 edge.json,验证空字段被 `init(from:)` 拒绝
   - 验证 `appScope` 值域越界被拒绝
   - 验证 `displayKey` 含非法字符被拒绝
   - 验证 `aliases` 归一后与 displayKey 冲突时拒启动

2. **Normalization 共享函数** (`Model/NormalizationTests.swift`)
   - `Cmd → ⌘`、`Shift → ⇧`、`Ctrl → ⌃`、`Option/Alt → ⌥`
   - 去空格和 `+`
   - 顺序: `⌃ ⌥ ⇧ ⌘` 固定
   - 大小写不敏感
   - 空输入、空格-only 输入

3. **状态机** (`Review/StateMachineTests.swift`)
   - 所有合法转移(D6 修订后的 7 条)
   - 防御性 `unseen → practiced` on 错误
   - 非法转移必须拒绝(例如 `unseen → mastered` 不允许)
   - 进入 mastered 时 `mistakeCount[id] = 0` 重置
   - recent_mistakes → mastered 答对 2 次的连续性

4. **4 选 1 干扰项生成** (`Review/DistractorGeneratorTests.swift`)
   - 同 `category + subcategory` 优先
   - 3 级 fallback(subcategory → category → 全库)
   - 排除正确答案
   - 3 个选项不重复
   - 选项顺序每次随机

5. **手输键判分** (`Review/KeyMatcherTests.swift`)
   - 大小写不敏感
   - 修饰键符号归一(用 Model/Normalization 共享函数)
   - aliases 归一也算对
   - 键位不一致判错
   - 判错时 `mistakeCount[id]` +1

6. **集成测试** (`Integration/AppModelEndToEndTests.swift`)
   - 加载 JSON → 勾选一条 → 状态变 practiced
   - 抽考出题 → 答对 → 状态变 mastered
   - 答错累计 2 次 → 状态变 recent_mistakes
   - 答对 2 次 → 状态变 mastered(跳过 practiced)

**不测:** UI、performance、兼容性。

**CI:** 本地 Xcode Test(`⌘U`)失败则 v1.0 不发布。**不是真正的 CI,只是单人 dogfood 的纪律。**

## Performance Review(本项目规模下结论)

- 200 条数据在内存中:trivially small(<1 MB)
- JSON 加载:同步、<10 ms
- 搜索 filter:O(n) on every keystroke,n=200,<1 ms
- @Observable 状态更新:SwiftUI 内部优化,实测无抖动
- **v1 无性能问题。** v2 如果条目增长到 1000+,需要 debounce 搜索或建索引。留作 TODO。

## Code Quality Predictions(无代码阶段的预测,落地时核对)

**P1(已锁定):**

## Failure Modes(每个 codepath 的失败场景)

| Codepath | 失败场景 | 现有测试 | 错误处理 | 用户可见错误? |
|----------|---------|---------|---------|--------------|
| 加载 JSON | bundle 缺失 / 格式错 / 字段越界 | 4 个 case | `init(from:)` throw,AppModel 拒启动 | 是,弹 "数据加载失败,联系开发者" |
| Normalization 冲突 | aliases 归一后与 displayKey 撞 | 1 个 case | 加载时打印 warning,拒启动 | 是,弹错 |
| UserDefaults 写失败 | 磁盘满 / 权限 | 无 | 暂未处理,降级到内存 | 否,状态丢失但 UI 看似正常 |
| 抽考候选为空 | 所有已练过条目都 mastered | 无 | UI 显示"今天没有可复习的" | 是,正面提示 |
| 4 选 1 干扰项不足 | category 极小(如 vscode < 4) | 3 级 fallback | 自动 fallback 到全库 | 是,可能看到跨 app 干扰项 |
| 状态机非法转移 | 状态机代码 bug 导致跳过状态 | 全部合法转移覆盖 | 状态机内部 throw,view 拒绝 | 取决于 throw 位置 |

**Critical gap:** UserDefaults 写失败无 silent fail——勾了但没存。需要补一个 `try?` 包装 + 错误日志。
